# Reglas de Desarrollo (Índice)

Propósito: evitar errores recurrentes y estandarizar cómo implementamos features en `backend-alquila` y `frontend-alquila`.

## Documentos vigentes

1. Reglas core (obligatorias): `docs/core-development-rules.md`
2. Issues conocidos y workarounds: `docs/known-issues.md`
3. Definition of Done técnica (checklist reusable): `docs/dod-tecnica.md`

## Uso recomendado

- Antes de arrancar un ticket: revisar `docs/dod-tecnica.md`.
- Durante implementación: aplicar `docs/core-development-rules.md`.
- Si aparece un bug de librería/proveedor: registrar en `docs/known-issues.md`.

## Nota de mantenimiento

Este índice reemplaza la versión monolítica previa para reducir ruido y facilitar enforcement en PR/CI.
