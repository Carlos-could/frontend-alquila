# Runbook de soporte (MVP)

Fecha: 2026-02-22
Estado: borrador

## SLA inicial
- Primera respuesta: < 12h.
- Resolución de incidencias críticas: < 24h.

## Clasificación
- P0: caída total o bloqueo de cierre/pago.
- P1: error funcional con workaround parcial.
- P2: incidencia menor o consulta operativa.

## Flujo operativo
1. Registrar ticket con ciudad, rol y evidencia.
2. Priorizar por severidad e impacto.
3. Escalar a ingeniería cuando aplica.
4. Cerrar ticket con causa raíz y acción preventiva.
